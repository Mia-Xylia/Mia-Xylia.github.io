# 类与对象

类是抽象的，概念的；对象是具体的，实际的
成员变量 = 属性 = field 字段
属性可以是基本数据类型，也可以是引用类型

内存布局

  • 细节
    1. 属性的定义 访问修饰符 属性类型 属性名；
    public protected 默认 private
    2. 属性如果不赋值，默认值规则和数组一样
    3. 属性的定义类型可以为任意类型，包含基本类型或引用类型

# 类与对象的内存分配机制

类与对象的内存分配机制
栈：一般存放基本数据类型（局部变量）
堆：存放对象（Cat cat, 数组）
方法区：常量池（常量，比如字符串），类加载信息

# 对象创建过程

1. 加载类信息（属性和方法信息，只会加载一次）
2. 在堆中分配空间，进行默认初始化
3. 把地址返回给栈中的对象名
4. 进行指定初始化

# 方法的好处

1. 提高代码的复用性
2. 可已经实现的细节封装起来，然后供其他用户来调用即可

# 方法的使用细节

  • 修饰符
    访问修饰符：控制方法使用的范围 public protected 默认 private
  • 返回值类型
    1. 一个方法最多有一个返回值（多个返回值可以返回数组）
    2. 返回类型可以是任何类型
    3. 如果方法要求有返回值类型，方法体中必须执行 return 语句，并且 return 值与返回值类型一致或兼容
    4. 如果方法为 void，则方法体中没有 return 语句，或者只写 return
    5. 驼峰命名法，见名知意
  • 形参列表
    1. 一个方法可以有 0 个参数，也可以有多个参数，中间用逗号隔开
    2. 参数类型可以为任意类型
    3. 调用带参数的方法时，一定对应着参数列表传入相同类型或者兼容类型
    4. 方法定义时的参数叫形参，调用方法时输入的参数叫实参，实参和形参类型要一致或者兼容，个数顺序均要一致
